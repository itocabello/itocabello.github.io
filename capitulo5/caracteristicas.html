4.3.1 Introducción al ecosistema Big Data

Este capítulo constituye el punto de partida de la formación práctica del proyecto y tiene como finalidad proporcionar al estudiante una comprensión sólida de los fundamentos de Big Data y de su aplicación sobre una infraestructura distribuida basada en Apache Hadoop.

La metodología seguida combina el estudio de los conceptos teóricos con su aplicación directa sobre el clúster Hadoop desplegado mediante tecnologías de virtualización ligera utilizando Docker.

El conocimiento de los principios de Big Data constituye la base necesaria para comprender el funcionamiento de las tecnologías distribuidas utilizadas durante el desarrollo de las prácticas del proyecto.

¿Por qué estudiar Big Data?

Comprender el paradigma Big Data resulta imprescindible para trabajar con infraestructuras distribuidas como la desarrollada en este proyecto. El clúster Hadoop implementado permite experimentar de forma práctica con las principales tecnologías del ecosistema Apache.

HDFS

Comprender cómo los datos se almacenan de forma distribuida y replicada entre los diferentes nodos del clúster.

YARN

Entender cómo se administran los recursos del clúster y cómo se distribuyen las tareas de procesamiento entre los distintos nodos.

MapReduce

Analizar el modelo de procesamiento paralelo utilizado para ejecutar aplicaciones sobre grandes volúmenes de información.

Objetivos de aprendizaje

Al finalizar este capítulo, el estudiante dispondrá de una base conceptual que le permitirá comprender tanto la arquitectura desplegada como el funcionamiento de las herramientas utilizadas durante el resto del proyecto.

Conocimientos adquiridos
  • Comprender el concepto de Big Data.
  • Conocer la importancia del procesamiento distribuido.
  • Identificar los principales componentes del ecosistema Hadoop.
  • Entender la relación entre teoría y práctica.
Competencias
  • Interpretar la arquitectura Hadoop desplegada.
  • Comprender el funcionamiento de HDFS.
  • Entender cómo YARN distribuye los recursos.
  • Relacionar MapReduce con el procesamiento paralelo.

Objetivo del capítulo

El propósito de este módulo consiste en proporcionar una base conceptual sólida sobre Big Data que permita al estudiante comprender cómo funcionan las infraestructuras distribuidas implementadas en el proyecto. Estos conocimientos servirán de apoyo para el desarrollo de las prácticas posteriores sobre el clúster Hadoop, facilitando la administración de los servicios, el almacenamiento de datos en HDFS y la ejecución de aplicaciones mediante MapReduce y Hadoop Streaming.
