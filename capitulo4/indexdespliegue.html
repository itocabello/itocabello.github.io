4 Arquitectura de Contenedores para el Despliegue del Entorno Big Data

Con el objetivo de disponer de un entorno distribuido, reproducible, escalable y sencillo de desplegar, la infraestructura utilizada en este proyecto se implementa mediante Docker y se orquesta utilizando Docker Compose. Esta arquitectura permite encapsular cada componente del ecosistema Hadoop en un contenedor independiente, simplificando tanto la instalación como la administración del clúster.

La utilización de contenedores permite desplegar un entorno Big Data completo en pocos minutos, facilitando su utilización tanto en entornos docentes como profesionales.

Objetivos del despliegue

Acceso abierto
La infraestructura se basa exclusivamente en tecnologías de código abierto, garantizando disponibilidad, documentación y soporte por parte de la comunidad.
Perfil del alumnado
La plataforma está diseñada para estudiantes de ingeniería y titulaciones técnicas, facilitando el aprendizaje práctico del ecosistema Hadoop.
Consumo de recursos
El despliegue puede ejecutarse sobre un ordenador personal con requisitos hardware moderados, permitiendo realizar prácticas sin necesidad de servidores dedicados.
Recomendación: Para un funcionamiento fluido se aconseja asignar al menos 1 GB de memoria RAM a la máquina virtual. Aunque es posible trabajar con menos memoria, el rendimiento disminuirá al desplegar servicios adicionales.

Arquitectura propuesta

Tras analizar las distintas alternativas existentes en los capítulos anteriores, se ha seleccionado una arquitectura basada en Docker y Docker Compose, cuyo objetivo es maximizar el aprovechamiento de los recursos hardware disponibles.

Toda la infraestructura se construye a partir de una imagen base común, desde la cual se generan los diferentes contenedores especializados que forman el clúster Hadoop.

Figura. Arquitectura propuesta basada en Docker y Docker Compose.

Docker Compose como herramienta de orquestación

El despliegue completo se realiza mediante un único fichero docker-compose.yml, encargado de automatizar la creación del entorno Big Data.

  • Creación automática de contenedores.
  • Configuración de la red privada.
  • Definición de dependencias entre servicios.
  • Persistencia mediante volúmenes Docker.
  • Inicio coordinado del clúster.
  • Escalabilidad sencilla.
  • Configuración centralizada.
  • Reproducibilidad del entorno.

Arquitectura del clúster Hadoop

La infraestructura desarrollada está formada por un conjunto de contenedores Docker que representan los distintos nodos del clúster Hadoop.

Componente Cantidad Función
NameNode 1 Gestión del sistema de archivos HDFS y metadatos.
DataNode 4 Almacenamiento distribuido de bloques de datos.
ResourceManager 1 Gestión global de recursos YARN.
NodeManager 4 Ejecución de tareas distribuidas.

Proceso de despliegue

1. Configuración del clúster
Se define la infraestructura mediante Docker Compose, permitiendo adaptar el número de nodos a los recursos disponibles.
2. Creación de contenedores
Cada contenedor representa un componente específico del ecosistema Hadoop, como NameNode, DataNode o ResourceManager.
3. Configuración de servicios
Los servicios se conectan automáticamente mediante una red privada Docker y se establecen las dependencias necesarias para el funcionamiento del clúster.
4. Persistencia de datos
Los datos se almacenan en HDFS utilizando volúmenes Docker para mantener la información incluso después de reiniciar los contenedores.

Ventajas de la arquitectura propuesta

  • Despliegue completamente automatizado.
  • Escalabilidad horizontal.
  • Portabilidad entre equipos.
  • Facilidad de mantenimiento.
  • Infraestructura reproducible.
  • Consumo reducido de recursos.
  • Separación entre servicios.
  • Ideal para laboratorios docentes.

Conclusión

La arquitectura propuesta combina Docker y Docker Compose para desplegar un clúster Hadoop completamente funcional mediante contenedores. Este enfoque proporciona una infraestructura ligera, escalable y fácilmente reproducible, permitiendo a los estudiantes centrarse en el aprendizaje de las tecnologías Big Data sin afrontar procesos de instalación complejos.
