Figura 19. Clúster Hadoop desplegado mediante Docker y Docker Compose.

Introducción

Una de las principales ventajas de utilizar Docker para implementar un clúster Hadoop es la elevada portabilidad y escalabilidad que ofrecen los contenedores. Cada servicio puede desplegarse de forma independiente, replicarse fácilmente y trasladarse entre diferentes equipos sin necesidad de repetir todo el proceso de instalación.

Con el objetivo de proporcionar un entorno de trabajo aislado, reproducible y fácilmente mantenible, la infraestructura desarrollada en este proyecto se despliega sobre una máquina virtual gestionada mediante VMware® Workstation 17 Pro. Esta solución permite ejecutar un sistema operativo independiente del equipo anfitrión, facilitando la instalación y administración de todas las herramientas que forman parte del ecosistema Big Data.

Virtualización tradicional frente a contenedores

En un escenario convencional, la construcción de un clúster Hadoop requiere múltiples máquinas virtuales, cada una con su propio sistema operativo, aplicaciones y dependencias instaladas de forma independiente. Este enfoque incrementa considerablemente el consumo de recursos y el tiempo necesario para desplegar el entorno.

En el laboratorio desarrollado para este proyecto se adopta una estrategia basada en virtualización ligera mediante contenedores Docker. Todos los servicios comparten una imagen base común que incorpora previamente las dependencias necesarias, reduciendo significativamente el consumo de memoria, el espacio en disco y el tiempo de despliegue.

Gracias a esta arquitectura, la creación de nuevos nodos consiste únicamente en replicar la imagen base y lanzar nuevos contenedores, facilitando enormemente la escalabilidad del clúster.

Arquitectura del clúster

El laboratorio está formado por un total de seis nodos virtuales:

  • 1 NameNode, encargado de administrar HDFS e incorporando Jupyter Notebook, Apache Hive y Apache Flume.
  • 1 YarnManager, responsable de la gestión de recursos del clúster y de coordinar la ejecución de aplicaciones distribuidas.
  • 4 DataNodes, donde se almacenan los bloques de datos de HDFS y se ejecutan las tareas MapReduce.

Descarga del proyecto

Todos los archivos necesarios para desplegar el laboratorio se encuentran publicados en un repositorio GitHub. El estudiante únicamente deberá descargar el proyecto completo y ejecutar Docker Compose para disponer de una infraestructura Big Data completamente operativa.

Resultado.

Mediante esta arquitectura basada en Docker se obtiene un entorno distribuido, portable, reproducible y fácilmente escalable, capaz de simular el funcionamiento de un clúster Hadoop real utilizando un único equipo físico.
