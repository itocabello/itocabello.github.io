4.2 Despliegue de la infraestructura

Una de las principales ventajas de utilizar Docker para implementar un clúster Hadoop es la elevada portabilidad y escalabilidad que ofrecen los contenedores. Estos pueden trasladarse fácilmente entre distintos equipos y replicarse de forma rápida para aumentar la capacidad del clúster sin necesidad de realizar configuraciones complejas.

Con el objetivo de proporcionar un entorno de trabajo aislado, reproducible y fácilmente distribuible, la infraestructura desarrollada en este proyecto se ha desplegado sobre una máquina virtual gestionada mediante VMware® Workstation 17 Pro. Esta solución permite ejecutar un sistema operativo invitado completamente independiente del sistema anfitrión, facilitando la instalación, configuración y mantenimiento de todas las herramientas que forman parte del ecosistema Big Data.

Toda la infraestructura puede trasladarse a cualquier equipo simplemente copiando la máquina virtual y ejecutando Docker Compose, garantizando un entorno totalmente reproducible.

Entorno de virtualización

El sistema anfitrión ejecuta Microsoft Windows 11, mientras que la máquina virtual utiliza Ubuntu 22.04.4 LTS, distribución seleccionada por su estabilidad, compatibilidad con Docker y soporte de largo plazo. Los recursos asignados han sido dimensionados para permitir la ejecución simultánea del clúster Hadoop y del resto de aplicaciones empleadas durante el desarrollo del proyecto.

Virtualización ligera mediante Docker

En una infraestructura Hadoop tradicional es necesario desplegar varias máquinas virtuales, cada una con su propio sistema operativo y con todas las dependencias necesarias para ejecutar los diferentes servicios del clúster. Este enfoque incrementa el consumo de recursos y dificulta la administración del entorno.

La arquitectura propuesta en este proyecto sustituye este modelo por una infraestructura basada en contenedores Docker, utilizando una única imagen base previamente configurada. A partir de ella se generan los distintos nodos del clúster, simplificando el despliegue, reduciendo el consumo de recursos y facilitando la escalabilidad del sistema.

El uso de Docker permite construir un laboratorio Hadoop completamente funcional sobre una única máquina virtual, manteniendo un entorno reproducible, portable y fácil de administrar mediante Docker Compose.

Conclusión

La arquitectura propuesta combina Docker y Docker Compose para desplegar un clúster Hadoop completamente funcional mediante contenedores. Este enfoque proporciona una infraestructura ligera, escalable y fácilmente reproducible, permitiendo a los estudiantes centrarse en el aprendizaje de las tecnologías Big Data sin afrontar procesos de instalación complejos.
