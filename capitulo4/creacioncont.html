4.2.2 Creación de contenedores

La creación de los contenedores constituye una de las fases fundamentales del despliegue de la infraestructura Big Data basada en Docker. Durante este proceso se construyen las imágenes necesarias y se instancian los distintos servicios que forman el clúster Hadoop, garantizando una configuración homogénea y una correcta comunicación entre todos los nodos.

Cada uno de los componentes del clúster dispone de su propio Dockerfile, encargado de definir las dependencias, librerías y configuración necesarias para su funcionamiento. Una vez situada la consola en el directorio correspondiente, la imagen puede construirse mediante el comando docker build.

Las imágenes utilizadas en este proyecto corresponden a hadoop-base, namenode, datanode, yarnmanager y db_cluster.

Construcción de una imagen Docker

Para generar una imagen Docker es suficiente con acceder al directorio donde se encuentra el correspondiente Dockerfile y ejecutar el siguiente comando:

Construcción de la imagen 
docker build -t nombre_imagen .

Donde nombre_imagen podrá corresponder a cualquiera de las imágenes que forman parte de la infraestructura, como por ejemplo:

  • hadoop-base
  • namenode
  • datanode
  • yarnmanager
  • db_cluster

Despliegue del clúster mediante Docker Compose

Una vez construidas las imágenes, no es necesario compilarlas nuevamente salvo que se realicen modificaciones sobre los Dockerfile. El despliegue completo del clúster se realiza desde el directorio raíz del proyecto, donde se encuentra el archivo docker-compose.yml.

Inicio del clúster Hadoop 
sudo docker-compose up -d

Este comando automatiza completamente el despliegue de la infraestructura, realizando las siguientes tareas:

  • Descarga las imágenes necesarias desde Docker Hub (si no existen localmente).
  • Crea automáticamente la red privada del clúster.
  • Genera e inicia todos los contenedores definidos en docker-compose.yml.
  • Configura las dependencias entre los distintos servicios.
  • Pone en marcha los servicios Hadoop y YARN.

Figura 24. Ejecución del comando docker-compose up -d para desplegar automáticamente el clúster Hadoop.

Resultado del despliegue

Tras ejecutar el comando anterior, Docker Compose descarga las imágenes necesarias, crea la red interna del clúster clusteruned_hadoop, inicializa los contenedores NameNode, DataNodes y YarnManager, y deja la infraestructura completamente operativa para comenzar las prácticas de Big Data.
