Docker Compose

Docker Compose es una herramienta que permite definir y ejecutar aplicaciones compuestas por múltiples contenedores Docker mediante un único archivo de configuración en formato YAML. De esta forma es posible desplegar toda una infraestructura con un solo comando.

Docker Compose simplifica la administración de aplicaciones distribuidas, automatizando la creación de contenedores, redes, volúmenes y dependencias.

¿Qué es Docker Compose?

Docker Compose permite definir todos los servicios que forman una aplicación en un único fichero denominado docker-compose.yml. Posteriormente, Docker Compose interpreta este archivo y crea automáticamente todos los contenedores necesarios.

Archivo YAML

Contiene la definición completa de la infraestructura:

  • Servicios
  • Imágenes
  • Redes
  • Volúmenes
  • Variables de entorno
  • Puertos
Despliegue

Toda la infraestructura puede iniciarse mediante un único comando. 

docker compose up -d

Ejemplo de archivo docker-compose.yml 

version: "3.9"
    
    services:
    
      namenode:
        image: apache/hadoop
        container_name: namenode
    
      datanode:
        image: apache/hadoop
    
      resourcemanager:
        image: apache/hadoop
    
      nodemanager:
        image: apache/hadoop

Este archivo describe los diferentes servicios que forman parte del clúster Hadoop y permite desplegarlos de forma automática.

Ventajas de Docker Compose

  • Un único archivo de configuración.
  • Despliegue mediante un solo comando.
  • Gestión automática de redes.
  • Gestión de volúmenes.
  • Fácil mantenimiento.
  • Infraestructura reproducible.
  • Escalado sencillo de servicios.
  • Ideal para laboratorios docentes.

Docker Compose en este Proyecto Fin de Grado

En este Proyecto Fin de Grado, Docker Compose se utiliza como mecanismo de orquestación para desplegar automáticamente el ecosistema Hadoop. Toda la infraestructura queda definida de forma declarativa en un único archivo YAML, eliminando la necesidad de crear manualmente cada contenedor.

Mediante Docker Compose se automatiza la creación de:

NameNode
DataNodes
ResourceManager
NodeManagers

Proceso de despliegue

Figura. Flujo de despliegue utilizando Docker Compose.

Paso Acción
1 Definir la infraestructura en docker-compose.yml.
2 Ejecutar docker compose up -d.
3 Docker crea la red y los volúmenes.
4 Se descargan las imágenes necesarias.
5 Se inician todos los servicios del clúster Hadoop.

Conclusión

Docker Compose constituye una herramienta fundamental para la automatización del despliegue de infraestructuras basadas en contenedores. En el contexto del Big Data, permite reproducir de forma sencilla y consistente un clúster Hadoop completo, reduciendo el tiempo de configuración, minimizando errores manuales y facilitando la realización de prácticas docentes y de investigación.
